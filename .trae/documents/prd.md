# 无下肢五分化健身打卡记录网站 - 产品需求文档

## 1. 产品概述
这是一个极简的个人健身记录单页网站，专为固定五分化上肢训练设计。用户无需下载APP，打开网页即可使用，每日记录训练重量、状态、完成情况，自动统计训练数据、生成训练复盘，精准跟踪上肢增肌进度。

**核心价值**：
- 100%匹配无下肢五分化训练计划
- 零学习成本，动作全部内置
- 极致轻量化，网页打开即用
- 本地存储，无需注册登录

## 2. 核心功能

### 2.1 用户角色
无角色区分，单一用户使用

### 2.2 功能模块
1. **首页**：训练日历总览、本周训练进度、快速入口
2. **训练打卡页**：当日训练详情、4个动作卡片、数据录入、教学视频
3. **历史记录页**：按日期查看历史训练数据
4. **数据复盘页**：训练统计、力量提升图表、薄弱部位分析
5. **设置页**：个人设置、训练计划说明、数据管理

## 3. 核心流程

### 3.1 训练打卡流程
用户打开网站 → 查看今日训练部位 → 点击进入训练打卡页 → 查看动作教学视频（可选）→ 逐组录入训练数据 → 系统自动保存 → 完成所有动作后点击完成打卡 → 查看当日训练总结

### 3.2 数据复盘流程
用户打开网站 → 点击数据复盘 → 查看训练完成率 → 查看力量提升图表 → 分析薄弱部位 → 查看PB记录

## 4. 用户界面设计

### 4.1 设计风格
- **主色调**：深色健身风主题（黑灰主色 #1a1a1a + 橙色强调色 #ff6b35）
- **按钮风格**：圆角按钮，带阴影和悬停效果
- **字体**：使用粗体标题字体突出训练部位，清晰易读的正文字体
- **布局风格**：卡片式布局，顶部导航，移动端优先
- **图标风格**：使用lucide-react图标库，简洁线性风格

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|---------|---------|---------|
| 首页 | 训练日历 | 网格布局日历，不同部位用不同颜色标记，橙色高亮今日，绿色标记已完成 |
| 首页 | 本周进度 | 进度条显示完成率，卡片式快速入口按钮 |
| 训练打卡页 | 动作卡片 | 白色卡片背景，圆角设计，每组数据用分隔线区分 |
| 训练打卡页 | 数据录入 | 数字输入框，下拉选择框，文本备注框 |
| 训练打卡页 | 计时器 | 大字体倒计时显示，开始/暂停/重置按钮 |
| 历史记录页 | 日期列表 | 时间轴式布局，每个日期卡片显示训练部位和完成状态 |
| 数据复盘页 | 图表区 | 使用Chart.js或Recharts绘制折线图、柱状图 |
| 设置页 | 设置项 | 列表式布局，开关按钮，确认对话框 |

### 4.3 响应式设计
- **移动端优先**：完美适配手机屏幕，健身房使用便捷
- **桌面端优化**：大屏查看月度训练数据、复盘图表
- **触控优化**：按钮和输入框足够大，方便触控操作

### 4.4 交互细节
- **一键填充**：下一组自动默认填充上一组重量
- **计时功能**：点击开始计时，倒计时结束弹窗提醒
- **完成校验**：所有动作完成才可点击完成按钮
- **数据对比**：突破个人最佳时自动提示
- **动画效果**：页面切换动画、按钮点击反馈、数据加载动画

## 5. 数据模型

### 5.1 训练计划数据
```typescript
interface TrainingDay {
  day: number;
  name: string; // 胸部日、背部日、肩部日、手臂日、核心日
  exercises: Exercise[];
}

interface Exercise {
  id: string;
  name: string;
  sets: number; // 固定4组
  reps: number; // 固定12次
  videoUrl?: string; // B站教学视频链接
}
```

### 5.2 训练记录数据
```typescript
interface WorkoutRecord {
  id: string;
  date: string; // YYYY-MM-DD
  dayType: string; // 胸/背/肩/手臂/核心/休息
  exercises: ExerciseRecord[];
  notes: WorkoutNotes;
  duration: number; // 总训练时长（分钟）
  completedAt: string; // 完成时间戳
}

interface ExerciseRecord {
  exerciseId: string;
  exerciseName: string;
  sets: SetRecord[];
}

interface SetRecord {
  setNumber: number;
  weight: number; // kg
  actualReps: number; // 实际完成次数
  restTime: number; // 组间休息时长（秒）
  status: 'normal' | 'failure' | 'exhausted' | 'good_form'; // 训练状态
  note?: string; // 小组备注
}

interface WorkoutNotes {
  bodyCondition: number; // 身体状态评分 1-10
  sleepQuality?: string; // 睡眠情况
  feeling?: string; // 训练体感
  supplements?: string; // 补剂
  diet?: string; // 饮食
  injury?: string; // 伤病情况
  warmup?: boolean; // 热身情况
  stretching?: boolean; // 拉伸情况
}
```

### 5.3 统计数据
```typescript
interface WorkoutStats {
  totalWorkouts: number; // 总训练次数
  weeklyCompletion: number; // 本周完成率
  monthlyCompletion: number; // 本月完成率
  personalBests: Record<string, number>; // 各动作PB记录
  progressData: ProgressPoint[]; // 力量提升数据
  weakAreas: string[]; // 薄弱部位
}

interface ProgressPoint {
  date: string;
  exerciseId: string;
  avgWeight: number;
}
```

## 6. 技术要求

### 6.1 性能要求
- 首屏加载时间 < 2秒
- 数据本地存储，离线可用
- 响应式设计，适配各种屏幕尺寸

### 6.2 兼容性要求
- 支持主流浏览器（Chrome、Firefox、Safari、Edge）
- 移动端浏览器完美适配
- 支持PWA特性（可选）

### 6.3 数据存储
- 使用LocalStorage存储所有训练数据
- 支持数据导出为Excel格式
- 数据自动备份机制
